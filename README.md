# Tourist Center CRM

Система управления туристическим центром на базе Laravel.

## Быстрый старт для разработчиков

Чтобы развернуть проект локально и начать работу, выполните следующие шаги:

### 1. Требования
Убедитесь, что у вас установлены:
- **PHP** (рекомендуется 8.2+)
- **Composer**
- **Node.js** и **NPM**
- **PostgreSQL** (убедитесь, что сервер запущен)

### 2. Установка зависимостей
Сначала установите PHP-пакеты:
```bash
composer install
```

Затем установите JavaScript-зависимости:
```bash
npm install
```

### 3. Настройка окружения
Скопируйте файл конфигурации:
```bash
cp .env.example .env
```

Сгенерируйте ключ приложения:
```bash
php artisan key:generate
```

### 4. Настройка базы данных
1. Откройте файл `.env`.
2. Укажите ваши данные для подключения к PostgreSQL:
   - `DB_DATABASE` (имя базы данных)
   - `DB_USERNAME` (имя пользователя)
   - `DB_PASSWORD` (пароль)

### 5. Миграции
Создайте структуру таблиц в базе данных:
```bash
php artisan migrate
```

### 6. Запуск проекта
Запустите сервер разработки Laravel:
```bash
php artisan serve
```

В другом окне терминала запустите сборку фронтенда (Vite):
```bash
npm run dev
```

---

## Разделение Frontend и Backend
В этом проекте фронтенд и бекенд четко разделены для удобства совместной работы:
- **`frontend/`** — папка для фронтенд-разработчиков. Здесь находятся React-компоненты, стили (SCSS) и статические ресурсы.
- **`app/`, `routes/`, `database/`** — папка для бекенд-разработчика.

### Инструкция для Frontend-разработчика
Если вы занимаетесь только интерфейсом, вам не нужно глубоко изучать Laravel.

1. **Установка**:
   ```bash
   npm install
   ```
2. **Запуск**:
   Запустите сборщик Vite:
   ```bash
   npm run dev
   ```
3. **Работа с файлами**:
   - Все исходники лежат в `frontend/js/` и `frontend/scss/`.
   - Если нужно изменить HTML-каркас, загляните в `resources/views/`.

*Примечание: Для корректной работы Vite всё равно может потребоваться запущенный `php artisan serve`, чтобы видеть результат на локальном домене.*
